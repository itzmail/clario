    - Test D-03: UID 0 process at /tmp/root_evil returns RootOutsideSystemPath; UID 0 at /usr/sbin/httpd does not
    - Test D-04: Process with cpu_usage 85.0 returns HighCpu flag; process with cpu_usage 50.0 does not
    - Test D-05: Process named "xmrig-worker" returns KnownBadName; process named "safari" does not
    - Test: Result is sorted suspicious-first (more flags first), then alphabetical
    - Test: Process with exe() == None does not trigger D-01, D-02, or D-03 (no false flags on SIP-protected processes)
  </behavior>
  <action>
    Create `src/core/process_scanner.rs` with:

    1. `pub struct ProcessScanner;` (stateless, like AppScanner pattern)

    2. `impl ProcessScanner` with `pub fn scan(sys: &sysinfo::System) -> Vec<ProcessInfo>`:
       - Iterate `sys.processes().values()`
       - For each process, extract: pid, name (`.name().to_string_lossy().to_string()`), exe (`.exe().map(|e| e.to_path_buf())`), cpu_usage, memory (`.memory()`), user_id (`.user_id().map(|u| **u)`), parent_pid (`.parent()`), run_time (`.run_time()`)
       - Apply 5 rules to build `Vec<SuspicionFlag>`:

       **D-01 (SuspiciousPath):** Only if exe is Some. Check exe_str does NOT start with any of: `/usr/`, `/bin/`, `/sbin/`, `/System/`, `/Applications/`, `/Library/`. If outside all trusted paths, push `SuspicionFlag::SuspiciousPath(exe_str.clone())`.

       **D-02 (NameExeMismatch):** Only if exe is Some. Compare `process.name().to_string_lossy()` with `Path::new(&exe_str).file_name().to_string_lossy()`. If they differ, push `SuspicionFlag::NameExeMismatch(format!("name='{}' exe='{}'", name, basename))`.

       **D-03 (RootOutsideSystemPath):** Only if exe is Some AND user_id == Some(0). Check exe_str does NOT start with `/System/`, `/usr/sbin/`, `/sbin/`. If outside system paths, push `SuspicionFlag::RootOutsideSystemPath(exe_str.clone())`.

       **D-04 (HighCpu):** If `cpu_usage > 80.0`, push `SuspicionFlag::HighCpu(cpu_usage)`. No exe requirement.

       **D-05 (KnownBadName):** Check `name.to_lowercase()` contains any of: `"xmrig"`, `"cryptonight"`, `"miner"`. Push `SuspicionFlag::KnownBadName(matched_string)`. Break after first match.

       - Construct `ProcessInfo` with `is_selected: false`
       - Sort result: `b.suspicion_flags.len().cmp(&a.suspicion_flags.len()).then(a.name.cmp(&b.name))`
       - Return `Vec<ProcessInfo>`

    3. Helper functions:
       - `fn format_memory(bytes: u64) -> String` — returns "X.XG" / "XM" / "XK" (used by UI later but co-located with scanner for testability)
       - `fn format_uptime(secs: u64) -> String` — returns "Xs" / "Xm Xs" / "Xh Xm"

    4. Add `pub mod process_scanner;` to `src/core/mod.rs`

    5. Unit tests in `#[cfg(test)] mod tests`:
       Since we cannot easily mock sysinfo::System, create helper functions that test the rule logic directly:
       - Extract rule checking into a standalone `fn apply_rules(name: &str, exe_path: Option<&str>, uid: Option<u32>, cpu: f32) -> Vec<SuspicionFlag>` that encapsulates the 5-rule logic. This makes unit testing straightforward without needing a real System instance.
       - `test_d01_suspicious_path`: exe="/tmp/evil" -> has SuspiciousPath
       - `test_d01_trusted_path`: exe="/usr/bin/ls" -> no SuspiciousPath
       - `test_d02_name_mismatch`: name="Safari", exe="/tmp/malware" -> has NameExeMismatch
       - `test_d02_name_match`: name="ls", exe="/usr/bin/ls" -> no NameExeMismatch
       - `test_d03_root_outside_system`: uid=0, exe="/tmp/evil" -> has RootOutsideSystemPath
       - `test_d03_root_in_system`: uid=0, exe="/usr/sbin/httpd" -> no RootOutsideSystemPath
       - `test_d04_high_cpu`: cpu=85.0 -> has HighCpu
       - `test_d04_normal_cpu`: cpu=50.0 -> no HighCpu
       - `test_d05_known_bad`: name="xmrig-worker" -> has KnownBadName
       - `test_d05_clean_name`: name="safari" -> no KnownBadName
       - `test_no_exe_no_path_rules`: exe=None -> no SuspiciousPath, no NameExeMismatch, no RootOutsideSystemPath flags
       - `test_format_memory`: 1073741824 -> "1.0G", 1048576 -> "1M", 1024 -> "1K"
       - `test_format_uptime`: 30 -> "30s", 90 -> "1m 30s", 3661 -> "1h 1m"
